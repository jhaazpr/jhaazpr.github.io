Collaborating on Taxon as an Undergraduate Researcher
I am working on a tool called Taxon that helps people who want to use 3D printers, laser cutters, or other digital fabrication machines find that right tool for the job. Taxon is a web application that hosts a database of machines, manufacturing tasks for machines, and rules for machine use that are all encoded as programs. Taxon lets users filter machines and safety check their tasks using these rules of thumb, which are user-defined lightweight program analyzers.
Right now, the Taxon interface is functional but not readily usable. It needs some additional features implemented, more example programs added and tested, and documentation written. This work combines a mix of interesting topics like:
  • front-end/full-stack web development
  • user interface design
  • human-computer interaction
  • programming languages research
  • digital fabrication machines
You can read more about Taxon on its project page here.
Please apply by emailing me at jaspero at cs dot washington dot edu. Please include a CV/resume, unofficial transcript, 3-4 sentences about why you're interested, and any solutions to the challenge problems below.
What Research Would Look like
You would be working on the Taxon interface as it is currently exists, building out new features and refactoring old ones. We'd start with small tasks and increase the scale as you become more comfortable with the code base. You would generally have a lot of agency as to what/how you'd implement things as you get the hang of the project's vision, which is useful for future cases where you need to describe a project you worked on (e.g. for a job or grad school applications).
I would expect you to devote around 5-10 hours a week towards research tasks, with flexibility allowed for when you have exams, job interviews, etc.
You would be paid in either research course credit or possibly be paid an hourly wage if my advisor can handle the costs at the moment.
Preparation
I would expect applicants to have completed CSE 311, 332, and 331 or have equivalent coursework or experience. Having taken CSE 340 or an equivalent course is a definite plus. Applicants do not need prior web programming experience, though some exposure is a plus. I prefer that applicants be interested in research for the sake of doing research, rather than looking to pad their resume.
Challenge Problems
Solving these problems is optional, but recommended if you have time. I recommend using Javascript/Typescript, but you can use any language if you really want. Please include any answer code when you email me.
Problem 1: Filtering Machine Plans
Write a function that takes an array of machine plans and returns an array of machine plans containing only machine plans in the input array which contain at least one block whose block type is "deltaBot". Machine plans are JSON objects whose structure is described in the BNF grammar on Taxon's project page.
For example, a machine plan has the form: 
{
  "name" : "myMachine",
  "metrics" : { ... },
  "blocks" : [
    {
      "name" : "myBlock",
      "dimensions" : { ... },
      "blockType" : "rotary"
    },
    ...
  ]
}
Problem 2: HTTP Requests and DOM Manipulation
Write a function that updates the HTML element on the page (whose id is #rule-of-thumb-names) with the names of all rules of thumb as obtained via an HTTP request to the server.
